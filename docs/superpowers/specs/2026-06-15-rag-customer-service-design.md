# RAG 客服对话系统 — 设计文档

## 项目概述

构建一套面向终端客户的客服对话 RAG 系统，支持多源文档检索与 LLM 生成回答，包含完整的管理后台与可观测性能力。

## 技术栈

| 层级 | 技术选型 |
|------|----------|
| 前端 | Vue 3 |
| 后端 | Python, FastAPI, LangChain 1.x |
| 向量数据库 | ChromaDB |
| 关系数据库 | PostgreSQL / SQLite |
| LLM | 火山引擎 API（豆包大模型） |
| 部署方式 | 混合部署（本地检索/存储 + 云端 LLM） |

## 核心功能模块

### 1. 文档处理流水线

- **格式解析**：支持 PDF、Word、Markdown、HTML 等常见格式
- **智能分块**：按文档结构（标题层级）分层切割，保持语义完整性
- **向量化入库**：LangChain 集成火山引擎 Embedding API，写入 ChromaDB

### 2. RAG 核心引擎

#### Query 处理模块
- **Query 重写**：对用户问题进行语义改写，提升检索召回率
- **指代消解**：多轮对话中解析"这个"、"它"等代词指向
- **历史压缩**：压缩多轮对话历史，避免上下文过长

#### 检索模块
- **混合检索**：向量检索 + BM25 关键词检索结合
- **重排序**：使用交叉编码器（cross-encoder）对初筛结果精排，保留 Top 3-5
- **检索融合**：统一打分，合并多路检索结果

#### 生成模块
- **Prompt 组装**：将检索结果、对话历史、系统指令组装为 LLM prompt
- **LLM 调用**：通过火山引擎 API 调用豆包大模型，支持流式输出
- **引用注入**：回答中标注来源文档片段编号/链接

### 3. API 网关层

- **客户对话 API**：用户提问 → 流式回答 → 返回引用来源
- **知识库管理 API**：文档上传、删除、更新、分类
- **后台管理 API**：日志查询、监控数据、参数配置
- **反馈收集 API**：用户点赞/点踩、修改建议提交

### 4. 可观测性模块

- **日志收集**：记录每次问答的 query、检索结果、LLM 回答、耗时
- **检索质量分析**：检索命中率、Top-K 准确率统计
- **用户反馈统计**：点赞/点踩率、按文档分类的满意度分布
- **告警**：异常检测（高延迟、空结果率异常升高等）

### 5. 前端模块

#### 客户对话界面
- 聊天窗口（消息列表 + 输入框）
- 流式回答展示（打字机效果）
- 来源引用展示（可点击查看原文片段）
- 反馈按钮（点赞/点踩 + 修改建议输入框）

#### 后台管理系统
- 文档上传/管理（上传、删除、更新、分类、预览）
- 问答日志查看（按时间、用户、文档分类筛选）
- 检索质量监控（仪表盘：调用量、平均延迟、检索命中率、满意度趋势）
- 参数配置（Top-K 数量、检索策略切换、重排序开关等）

## 数据模型

### 文档（Document）
- id, title, type, category, file_path, status, created_at, updated_at

### 文档块（Chunk）
- id, document_id, content, embedding_id, metadata, page_number, chunk_index

### 对话日志（Conversation Log）
- id, session_id, query, rewritten_query, retrieved_chunks, answer, feedback, latency, created_at

### 用户反馈（Feedback）
- id, conversation_id, rating (like/dislike), suggestion, created_at

## 开发路线

1. **Phase 1** — 基础 RAG 管道：文档解析 → 分块 → 向量化 → 检索 → LLM 回答（流式输出）
2. **Phase 2** — 检索增强：Query 重写、混合检索、重排序
3. **Phase 3** — 管理后台：文档管理、日志查看、参数配置
4. **Phase 4** — 可观测性：质量监控仪表盘、反馈收集、告警

## 数据量估算

- 文档数量：约 1,000 份
- ChromaDB 向量规模：按每份文档 10-50 个 Chunk 估算，约 10,000-50,000 条向量记录
- 此规模下单机部署 ChromaDB 完全足够，无需分布式方案
