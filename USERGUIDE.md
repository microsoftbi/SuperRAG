## 1. 系统概述

SPRAG 是一套面向终端客户的智能客服问答系统，基于 RAG（检索增强生成）架构。系统包含两个主要界面：

- **客服对话界面** (`/`) — 客户提问，AI 基于知识库回答，支持流式打字机效果
- **管理后台** (`/admin`) — 管理员管理文档、查看日志、监控系统状态、配置参数

### 技术架构

```
用户提问 → Query重写 → 混合检索(向量+BM25) → 重排序 → LLM生成回答
                    ↘ 来源引用标注 ↙
```

## 2. 快速启动

### 2.1 环境要求

- Python 3.11+
- Node.js 18+
- 火山引擎方舟平台 API 密钥

### 2.2 配置

```bash
# 1. 后端配置
cd backend
cp .env.example .env
```

编辑 `.env` 文件，填入以下信息：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| `VOLC_API_KEY` | 火山引擎 API 密钥 | `4abb27fb-xxxx` |
| `VOLC_ENDPOINT` | API 地址 | `https://ark.cn-beijing.volces.com/api/v3` |
| `LLM_MODEL_NAME` | 对话模型 ID | `DeepSeek-V4-Flash` |
| `EMBEDDING_MODEL_NAME` | Embedding 模型 ID | `doubao-embedding-vision` |

### 2.3 启动

```bash
# 终端 1：启动后端（端口 8000）
cd backend
pip install -r requirements.txt
python run.py

# 终端 2：启动前端（端口 5173）
cd frontend
npm install
npm run dev
```

启动后访问：
- 客服对话：`http://localhost:5173`
- 管理后台：`http://localhost:5173/admin`

## 3. 客服对话界面

### 3.1 发起对话

打开浏览器访问 `http://localhost:5173`，进入对话界面。

1. 在底部输入框输入问题
2. 按 Enter 或点击「发送」按钮
3. AI 会以打字机效果逐字流式输出回答

### 3.2 来源引用

回答中的 `[来源1]`、`[来源2]` 标记是可点击的引用来源：

- 点击 `[来源1]` 可展开查看对应的文档片段
- 来源卡片显示：文档标题、相关度分数、片段预览

### 3.3 反馈评价

每条 AI 回答底部有 👍/👎 按钮：

- **👍 有帮助** — 标记该回答有帮助
- **👎 没帮助** — 标记该回答没有帮助
- 反馈会记录到管理后台的「用户反馈」页面

### 3.4 多轮对话

系统支持多轮连续对话：
- 历史消息自动带入上下文
- 后续问题支持代词指代（如"它的价格是多少"→ 自动补充为完整问题）
- 当前会话刷新页面后丢失（会话历史持久化功能即将上线）

## 4. 管理后台

访问 `http://localhost:5173/admin` 进入管理后台，包含 5 个标签页：

### 4.1 仪表盘

系统运行状态概览，默认首页。

#### 概览卡片

| 指标 | 说明 |
|------|------|
| 总对话数 | 系统上线以来的总问答次数 |
| 今日对话 | 今天发起的对话次数 |
| 平均延迟 | 每次问答的平均响应时间（毫秒） |
| P95 延迟 | 95% 的请求都在此延迟以内 |
| 空结果率 | AI 无法回答（答案很短）的比例 |
| 总 Token 数 | LLM 累计消耗的 Token 总量 |
| 满意度 | 点赞数 / 总反馈数的比例 |
| 总反馈数 | 用户点击 👍/👎 的总次数 |

#### 告警提示

当系统出现异常时，顶部会显示告警横幅：

| 告警类型 | 触发条件 |
|----------|----------|
| ⚠️ 高延迟 | 最近 30 分钟平均延迟超过 10 秒 |
| ⚠️ 高空结果率 | 最近 30 分钟空结果率超过 50% |
| 🚨 低满意度 | 最近 30 分钟满意度低于 30% |

#### 趋势图表

三个柱状图展示近 7 天的变化趋势：
- **每日调用量** — 每天问答次数
- **平均延迟** — 每天平均响应时间
- **满意度** — 每天用户满意度变化

### 4.2 文档管理

知识库文档的增删管理。

#### 支持的文件格式

| 格式 | 扩展名 |
|------|--------|
| PDF | `.pdf` |
| Word | `.docx` |
| Markdown | `.md` |
| HTML | `.html`, `.htm` |
| 纯文本 | `.txt` |

#### 上传文档

1. 在「文档管理」标签页，点击「选择文件」
2. 可选填写「文档标题」和「分类」
3. 点击「上传」按钮
4. 上传后自动进入处理流程：
   - `待处理` → `处理中` → `就绪`（或 `失败`）

#### 文档状态说明

| 状态 | 含义 |
|------|------|
| 🟡 待处理 | 文件已保存，等待处理 |
| 🟠 处理中 | 正在解析、分块、向量化 |
| 🟢 就绪 | 处理完成，可被检索 |
| 🔴 失败 | 处理出错（格式问题或文件损坏） |

#### 删除文档

点击文档行尾的「删除」按钮即可移除文档及其所有向量数据。

#### 处理流程说明

上传文档后系统自动执行：
1. **文本提取** — 根据文件格式解析文本内容
2. **智能分块** — 按段落、句子分割为语义完整的块（默认每块 500 字符）
3. **向量化** — 调用 Embedding 模型转为向量，存入 ChromaDB
4. **关键词索引** — 重建 BM25 关键词索引

### 4.3 问答日志

记录每次问答的详细信息。

#### 日志列表

每条日志显示：
- 会话 ID（前 20 位）
- 提问时间
- 响应延迟（毫秒）
- 是否经过 Query 重写标记

#### 查看详情

点击日志卡片展开详情：
- **用户问题** — 原始提问
- **改写后查询** — Query 重写后的检索用查询（如有改写）
- **AI 回答** — 完整回答内容（前 500 字符）
- **来源数量** — 检索到的参考文档数
- **Token 数** — 回答消耗的 Token 量

#### 筛选

在顶部输入框输入「会话 ID」可筛选特定会话的全部日志。

### 4.4 用户反馈

查看用户对 AI 回答的评价。

#### 统计卡片

| 指标 | 说明 |
|------|------|
| 总反馈 | 用户点击反馈的总次数 |
| 点赞 | 标记"有帮助"的次数 |
| 点踩 | 标记"没帮助"的次数 |
| 满意度 | 点赞数占总反馈数的百分比 |

#### 反馈列表

每条反馈显示：
- 👍/👎 评价
- 提交时间
- 会话 ID
- 用户提交的建议文本（如有）

### 4.5 参数配置

运行时调整 RAG 检索参数，保存后立即生效（无需重启服务）。

#### 检索策略

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 检索策略 | 混合检索 / 纯向量检索 | 混合检索 |
| Query 重写 | 开启/关闭 | 开启 |
| 重排序 | 开启/关闭 | 开启 |

#### 数量参数

| 参数 | 说明 | 默认值 | 建议范围 |
|------|------|--------|----------|
| 向量检索数量 | 每次检索返回的向量结果数 | 10 | 5-30 |
| BM25 检索数量 | 关键词检索返回的结果数 | 10 | 5-30 |
| 重排序保留数量 | 精排后保留的结果数 | 5 | 3-10 |

#### LLM 参数

| 参数 | 说明 | 默认值 | 建议范围 |
|------|------|--------|----------|
| 温度 | 回答随机性，越高越有创造力 | 0.7 | 0.1-1.0 |

#### 分块参数

| 参数 | 说明 | 默认值 | 建议范围 |
|------|------|--------|----------|
| Chunk 大小 | 文档分块的字符数 | 500 | 200-1000 |

> **注意**：Chunk 大小仅对新上传的文档生效，已有文档不受影响。

## 常见问题

### Q: 上传文档后提示失败怎么办？

可能原因：
- 文件格式不支持（仅支持 PDF/DOCX/MD/HTML/TXT）
- 文件内容为空
- PDF 为扫描件（无文字层，需 OCR 预处理）

### Q: AI 回答不准确怎么办？

- 检查知识库是否包含相关文档
- 在「参数配置」中开启「混合检索」和「重排序」
- 适当降低 LLM 温度（如 0.3-0.5）
- 确保文档内容清晰、结构完整

### Q: 回答速度慢怎么办？

- 在「参数配置」中减少「向量检索数量」
- 关闭「重排序」功能
- 检查网络到火山引擎的延迟

### Q: 数据存在哪里？

- 向量数据：`backend/chroma_db/` 目录
- 关系数据：`backend/sprag.db`（SQLite）
- 上传文档：`backend/uploads/` 目录
- 运行时配置：`backend/runtime_config.json`
