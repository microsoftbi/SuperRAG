## 1. 系统概述

SuperRAG 是一套面向终端客户的智能客服问答系统，基于 **RAG + 知识图谱 + NL2SQL** 三引擎架构。

### 三引擎架构

```
┌─────────────────────────────────────────────────────────┐
│                    客服对话界面 (三 Tab)                    │
│  ┌──────────┐  ┌──────────┐  ┌──────────────────┐       │
│  │ [RAG]    │  │ [知识图谱]│  │ [问数]           │       │
│  │ 文档问答  │  │ 实体关系  │  │ 数据查询+图表     │       │
│  └────┬─────┘  └────┬─────┘  └──────┬───────────┘       │
└───────┼─────────────┼───────────────┼───────────────────┘
        │             │               │
        ▼             ▼               ▼
┌──────────────┐ ┌──────────┐ ┌──────────────────────┐
│ RAG 引擎     │ │ KG 引擎  │ │ NL2SQL 引擎           │
│ ┌──────────┐ │ │ ┌──────┐ │ │ ┌──────────────────┐ │
│ │ Milvus   │ │ │ │Neo4j │ │ │ │ 提示词动态拼装    │ │
│ │ 稠密+稀疏 │ │ │ │图遍历│ │ │ │ Schema + 术语    │ │
│ │ BM25 中文 │ │ │ │关系  │ │ │ │ + Q&A 示例       │ │
│ │ 批量重排序│ │ │ │文本  │ │ │ │ → LLM 生成 SQL   │ │
│ └──────────┘ │ │ └──────┘ │ │ │ → 执行 SQL Server │ │
└──────────────┘ └──────────┘ │ │ → 表格+ECharts图  │ │
                              │ └──────────────────┘ │
                              └──────────────────────┘
```

- **RAG 引擎**：文档分块 → 向量化(Milvus Lite) + BM25(language="zh") → 混合检索 → 批量 LLM 重排序 → 生成回答。适合"是什么/怎么做/多少钱"等事实型问题。
- **KG 引擎**：实体提取(LLM) → 写入Neo4j → 图遍历 → 返回关系文本。适合"谁负责/什么关系/属于哪个部门"等关系型问题（不再关联文档原文）。
- **NL2SQL 引擎**：独立 deepagents Agent → 动态拼装提示词（字段结构+专业术语+Q&A示例）→ LLM 生成 SQL Server 兼容 SQL → pyodbc 执行 → 格式化表格 + ECharts 图表。适合"销售额前10的客户"等数据查询问题。

### 技术架构

- 后端：Python, FastAPI, SQLAlchemy, Neo4j, Milvus Lite, pyodbc + FreeTDS
- Agent 框架：deepagents v0.5+（三 Agent：RAG/KG/NL2SQL，独立 checkpointer）
- 前端：Vue 3, Vite, vis-network（图谱可视化）, ECharts（图表）
- LLM：火山引擎 API（豆包大模型 / DeepSeek）
- 存储：SQLite（结构化数据）、Milvus Lite（向量+BM25稀疏索引）、Neo4j（图谱）、文件系统（文档原件）

## 2. 快速启动

### 2.1 环境要求

- Python 3.11+
- Node.js 18+
- 火山引擎方舟平台 API 密钥
- Neo4j 数据库（图谱引擎，可选）

### 2.2 安装 Neo4j（可选）

```bash
# macOS
brew install neo4j
brew services start neo4j

# 访问 http://localhost:7474 首次登录
# 默认账号: neo4j / neo4j（首次需修改密码）
```

### 2.3 配置

```bash
cd backend
cp .env.example .env
```

编辑 `.env` 文件，填入以下信息：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| `VOLC_API_KEY` | 火山引擎 API 密钥 | `4abb27fb-xxxx` |
| `VOLC_ENDPOINT` | API 地址 | `https://ark.cn-beijing.volces.com/api/v3` |
| `LLM_MODEL_NAME` | 对话模型 ID | `ep-xxxx-xxxxx` |
| `EMBEDDING_MODEL_NAME` | Embedding 模型 ID | `ep-xxxx-xxxxx` |
| `NEO4J_URI` | Neo4j 连接地址 | `bolt://localhost:7687` |
| `NEO4J_USER` | Neo4j 用户名 | `neo4j` |
| `NEO4J_PASSWORD` | Neo4j 密码 | `your_password` |

### 2.4 启动

```bash
# 终端 1：启动后端（端口 8000）
cd backend
pip install -r requirements.txt
python run.py

# 终端 2：启动前端（端口 5173）
cd frontend
npm install
npm run dev
```

启动后访问：
- 客服对话：`http://localhost:5173`
- 管理后台：`http://localhost:5173/admin`

### 2.5 默认管理员账号

| 用户名 | 密码 | 角色 |
|--------|------|------|
| `admin` | `admin123` | 管理员 |

## 3. 客服对话界面

### 3.1 三 Tab 模式

打开浏览器访问 `http://localhost:5173`，对话页面顶部有三个 Tab：

| Tab | 用途 | 底层引擎 | 会话 ID 前缀 |
|-----|------|---------|-------------|
| **RAG** | 文档知识问答 | deepagents (search_knowledge_base) | `session_` |
| **知识图谱** | 实体关系查询 | deepagents (search_knowledge_graph) | `kg_` |
| **问数** | 业务数据查询+图表 | deepagents (query_database + make_chart) | `nl2sql_` |

三个 Tab 的聊天历史、会话侧边栏完全隔离，切换 Tab 不会影响彼此。

> 若 Neo4j 未连接，「知识图谱」Tab 自动变灰不可点击。

### 3.2 发起对话

1. 在底部输入框输入问题
2. 按 Enter 或点击「发送」按钮
3. AI 会以打字机效果逐字流式输出回答

#### RAG Tab 示例

| 问题类型 | 示例 | 回答特点 |
|---------|------|---------|
| **文档检索** | "退款流程是什么？""产品的保修期多久？" | 显示 [来源X] 引用 |

#### 知识图谱 Tab 示例

| 问题类型 | 示例 | 回答特点 |
|---------|------|---------|
| **实体关系** | "库克跟苹果公司的关系？""A项目谁负责？" | 显示 ⛓ 图谱 + Cypher 按钮（无文档原文） |

#### 问数 Tab 示例

| 问题 | 回答特点 |
|------|---------|
| "销售额前10的客户" | 显示 SQL → 格式化表格（序号、千分位、tooltip） |
| "上个月的订单总数" | 显示 SQL → 数值结果 + 中文解读 |
| "画个柱图/饼图" | 在表格基础上生成 ECharts 图表 |

### 3.3 会话侧边栏

左侧侧边栏显示历史会话列表：

| 功能 | 说明 |
|------|------|
| **会话列表** | 按最后活跃时间倒序，显示首条问题和消息数 |
| **点击切换** | 点击历史会话，加载完整对话记录 |
| **新对话** | 点击「新对话」按钮创建空白会话 |
| **折叠/展开** | 侧边栏可点击折叠，扩大对话区域 |
| **重命名/删除** | 鼠标悬停会话行显示 ✏️/🗑️ 按钮 |

会话列表按当前 Tab 过滤：
- [RAG] Tab → 只显示 RAG 会话
- [知识图谱] Tab → 只显示知识图谱会话
- [问数] Tab → 只显示 NL2SQL 会话

### 3.4 消息气泡

#### 普通消息
- **用户消息**：蓝色气泡，右对齐
- **AI 消息**：灰色气泡，左对齐，流式打字机效果

#### 问数结果表格
问数消息包含 SQL 和结果数据，自动渲染为格式化表格：

```
┌─────────────────────────────────────┐
│ 共 10 行               [📋 复制CSV]  │
├─────────────────────────────────────┤
│  # │ 客户名称          │ 总销售额    │
│────│───────────────────│────────────│
│  1 │ Nicholas Nara     │   13,295.38│
│  2 │ Kaitlyn           │   12,894.62│
│  ⋮ │ ⋮                 │ ⋮          │
├─────────────────────────────────────┤
│  ↑ 可水平/垂直滚动                   │
└─────────────────────────────────────┘
```

- 数字列自动右对齐 + 千分位格式化
- 空值显示为 `—`
- 超长文字 hover 显示完整内容（tooltip）
- 支持水平滚动（宽表格不挤压）
- 「复制CSV」按钮一键导出

#### 问数图表
当用户要求"画图"时，系统调用 `make_chart` 工具生成 ECharts 图表：

- 支持类型：柱状图 / 饼图 / 折线图 / 面积图 / 散点图 / 堆叠柱状图
- 图表和表格共存，可切换图表类型

### 3.5 来源引用

RAG 回答底部显示来源引用标签：

- **RAG 来源**：`[1] 文档名 85%` — 显示文档名称和匹配度

### 3.6 知识图谱弹窗

知识图谱回答底部显示：

- `⛓ 图谱` 按钮 — 点击弹出 vis-network 力导向图，展示实体关系
- `Cypher` 按钮 — 点击显示本次查询的 Cypher 语句，可复制使用

图谱弹窗功能：
- **种子节点**：红色星形高亮（回答直接涉及的实体）
- **普通节点**：按类型着色（人物=红、组织=蓝、产品=绿）

### 3.7 反馈评价

每条 AI 回答底部有 👍/👎 按钮：

- **👍 有帮助** — 标记该回答有帮助
- **👎 没帮助** — 标记该回答没有帮助
- 反馈会记录到管理后台的「用户反馈」页面

### 3.8 多轮对话

系统支持多轮连续对话：
- 历史消息自动带入上下文（deepagents checkpointer 持久化）
- 会话历史刷新页面后保留
- 可通过侧边栏切换到任意历史会话

### 4.2 文档管理

知识库文档的增删管理。

#### 支持的文件格式

| 格式 | 扩展名 |
|------|--------|
| PDF | `.pdf` |
| Word | `.docx` |
| Markdown | `.md` |
| HTML | `.html`, `.htm` |
| 纯文本 | `.txt` |

#### 上传文档

上传文档时需选择**存储目标**：

| 存储目标 | 说明 | 处理流程 |
|---------|------|---------|
| **向量数据库** | 全文检索、语义搜索 | 分块 → 向量化(Milvus Lite) → BM25 索引 |
| **图数据库** | 实体关系提取 | LLM 提取实体/关系 → 写入 Neo4j |
| **全部** | 两个数据库都存储 | 同时执行以上两个流程 |

**操作步骤**：
1. 在「文档管理」标签页点击「选择文件」
2. 选择**存储目标**（向量数据库 / 图数据库 / 全部）
3. 选择**知识库**（仅向量数据库和全部需要）
4. 点击「上传」

#### 文档子标签

文档列表分两个子标签：

- **向量存储** — 显示存储到向量数据库的文档
- **图谱存储** — 显示存储到图数据库的文档

#### 查看文档分块

在「向量存储」子标签下，每个文档末尾有一个「查看分块」按钮：

点击后弹出对话框，左侧显示文档原文，右侧显示分块结果列表。每个分块显示块序号和内容摘要，帮助理解文档是如何被拆分成检索单元的。

#### 文档状态说明

| 状态 | 含义 |
|------|------|
| 🟡 待处理 | 文件已保存，等待处理 |
| 🟠 处理中 | 正在解析、分块、向量化或实体提取 |
| 🟢 就绪 | 处理完成，可被检索 |
| 🔴 失败 | 处理出错（格式问题或文件损坏） |

#### 删除文档

点击文档行尾的「删除」按钮即可移除文档及其所有关联数据。

#### 处理流程说明

**向量数据库流程**：
1. **文本提取** — 根据文件格式解析文本内容
2. **智能分块** — 按段落、句子分割为语义完整的块（默认每块 500 字符）
3. **向量化** — 调用 Embedding 模型转为向量，存入 Milvus Lite（稠密+稀疏双向量）
4. **关键词索引** — 重建 BM25 索引（Milvus BM25EmbeddingFunction, language="zh"）

**图数据库流程**：
1. **文本提取** — 根据文件格式解析全文内容
2. **LLM 实体提取** — 调用大模型提取实体（人物/组织/产品/概念/地点）和关系
3. **存入 Neo4j** — 实体作为节点，关系作为边，写入图数据库

### 4.5 NL2SQL 配置

问数功能的数据库连接和提示词配置面板。

#### 数据库连接

配置 SQL Server 连接信息：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| 连接名称 | 标识名称 | 测试库 |
| 主机地址 | SQL Server IP | 192.168.0.116 |
| 端口 | SQL Server 端口 | 1433 |
| 用户名 | 数据库账号 | sa |
| 密码 | 数据库密码 | Passw0rd |
| 数据库 | 数据库名 | AdventureWorksDW2016 |

填写后点击「测试连接」验证连通性。

#### 提示词配置（三个 Tab）

NL2SQL 功能的提示词由三部分组成，通过 LLM 动态拼装后生成 SQL：

| Tab | 用途 | 内容示例 |
|-----|------|---------|
| **字段结构** | 数据库表和字段定义 | 表名、字段名、类型、主外键 |
| **专业术语** | 业务术语到数据库字段的映射 | "销售额"="SalesAmount" |
| **Q&A示例** | 几个典型问答对，引导 LLM 生成正确 SQL | Q: 销售额前10客户 → A: SELECT TOP 10... |

> 提示词拼接规则：`你是一个数据分析师。数据库字段结构：{字段结构} 专业术语映射：{专业术语} 参考问答示例：{Q&A示例} 要求：...`

### 4.7 问答日志

记录每次问答的详细信息。

#### 日志列表

每条日志显示：
- 会话 ID（前 20 位）
- 提问时间
- 路由类型（AGENT / KG / NL2SQL）
- 响应延迟（毫秒）
- 是否经过 Query 重写标记

#### 查看详情

- **用户问题** — 原始提问
- **改写后查询** — Query 重写后的检索用查询（如有改写）
- **AI 回答** — 完整回答内容（前 500 字符）
- **来源数量** — 检索到的参考文档数
- **Token 数** — 回答消耗的 Token 量
- **生成的 SQL** — 仅 NL2SQL 类型日志，显示 LLM 生成的 SQL 语句（附带复制按钮）
- **SQL 提示词** — 仅 NL2SQL 类型日志，显示发给 LLM 的完整提示词（附带复制按钮）

> NL2SQL 的 SQL 和提示词是否写入日志，受「参数配置」→「明细日志」开关控制，默认关闭。

#### 清空日志

标题栏右侧「清空日志」按钮可删除所有日志记录（需确认，不可撤销）。

#### 筛选

在顶部输入框输入「会话 ID」可筛选特定会话的全部日志。

### 4.9 参数配置

运行时调整检索参数，保存后立即生效（无需重启服务）。

#### 检索策略

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 检索策略 | 混合检索 / 纯向量检索 | 混合检索 |
| Query 重写 | 开启/关闭上下文改写 | 开启 |
| 重排序 | 开启/关闭精排 | 开启 |

#### 数量参数

| 参数 | 说明 | 默认值 | 建议范围 |
|------|------|--------|----------|
| 向量检索数量 | 每次检索返回的向量结果数 | 10 | 5-30 |
| BM25 检索数量 | 关键词检索返回的结果数 | 10 | 5-30 |
| 重排序保留数量 | 精排后保留的结果数 | 5 | 3-10 |

#### LLM 参数

| 参数 | 说明 | 默认值 | 建议范围 |
|------|------|--------|----------|
| 温度 | 回答随机性，越高越有创造力 | 0.7 | 0.1-1.0 |

#### 分块参数

| 参数 | 说明 | 默认值 | 建议范围 |
|------|------|--------|----------|
| Chunk 大小 | 文档分块的字符数 | 500 | 200-1000 |

> **注意**：Chunk 大小仅对新上传的文档生效，已有文档不受影响。

#### 知识图谱

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 知识图谱遍历深度 | 图遍历的最大跳数，值越大检索越广但响应时间越长 | 5（范围 2-6） |

#### NL2SQL 行数限制

| 参数 | 说明 | 默认值 |
|------|------|--------|
| NL2SQL 最大行数 | 查询返回的最大记录数（LLM 自动注入 TOP N） | 50（范围 50-1000） |

#### 明细日志

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 明细日志 | 勾选后问数功能的 SQL 和提示词写入问答日志 | 未勾选 |

> 为保护敏感信息，SQL 语句和数据查询提示词默认不写入日志。需要排查问题时可临时开启。

## 5. 常见问题

### Q: 上传文档后提示失败怎么办？

可能原因：
- 文件格式不支持（仅支持 PDF/DOCX/MD/HTML/TXT）
- 文件内容为空
- PDF 为扫描件（无文字层，需 OCR 预处理）

### Q: AI 回答不准确怎么办？

- 检查知识库是否包含相关文档
- 在「参数配置」中开启「混合检索」和「重排序」
- 适当降低 LLM 温度（如 0.3-0.5）
- 确保文档内容清晰、结构完整

### Q: 问答页面没有图谱按钮？

- 确认 Neo4j 服务已启动：`brew services start neo4j`
- 确认后端启动日志有 `✅ Neo4j connected and initialized`
- 确认文档已上传到「图数据库」或「全部」
- 在知识图谱页面点「重建图谱」

### Q: 回答速度慢怎么办？

- 在「参数配置」中减少「向量检索数量」
- 关闭「重排序」功能
- 检查网络到火山引擎的延迟
- 减少知识图谱遍历深度（默认 5 → 2-3）

### Q: 问数功能连不上数据库怎么办？

- 确认 SQL Server 端口（默认 1433）可达：`nc -zv host 1433`
- 检查 NL2SQL 配置页面的账号密码（可点「测试连接」验证）
- 确认 SQL Server 已启用 TCP/IP 连接
- macOS 需安装 FreeTDS：`brew install freetds`

### Q: 问数返回的表格无法滚动/格式错乱？

- 多列表格支持水平滚动，在表格区域左右拖动
- 数字列自动右对齐并显示千分位
- 超长文字 hover 显示完整内容
- 可通过「复制CSV」按钮导出到 Excel

### Q: 数据存在哪里？

| 数据类型 | 存储位置 |
|---------|---------|
| 文档原件 | `backend/uploads/` |
| 结构化数据 | `backend/sprag.db`（SQLite） |
| 向量数据 | `backend/milvus_lite.db`（Milvus Lite） |
| BM25 稀疏索引 | Milvus Lite 同一集合（SPARSE_FLOAT_VECTOR 字段） |
| 图谱数据 | Neo4j（bolt://localhost:7687） |
| 运行时配置 | `backend/runtime_config.json` |
| NL2SQL 配置 | `backend/nl2sql_config.json` |
| 文档分块 | SQLite `chunks` 表 |
